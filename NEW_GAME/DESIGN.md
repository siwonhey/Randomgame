# 메탈블레이드 팽이 서바이벌 배틀 — 디자인 가이드

---

## 1. 레이아웃 구성

### 원칙: 올인원 싱글 스크린
- 입력, 실행, 결과가 **한 화면 안에서** 모두 이루어짐 (페이지 이동/모달 전환 없음)
- 미니멀하고 여백 중심의 구성 (참고: [Max Yinger](https://yinger.dev/?ref=godly) 스타일)

### 화면 배치 (데스크탑 기준)
```
┌─────────────────────────────────────────────────────┐
│  (헤더: 이벤트 제목 — 두꺼운 고딕체 )                     │
├────────────┬────────────────────────────────────────┤
│                                                     │
│                       GAME AREA                     │
│                     (3D 스타디움)                     │
│                                                     │
│                팽이들이 배치·충돌·탈락하는                 │
│                메인 렌더링 영역                         │
│                                                     │
│                                                     │
│                                                     │
│                                                     │
│  ──────── │                                         │
│     INPUT  │                               탈락순위     │     PANEL                                  (실시간)
│  -참가자 입력 │                                        │
│  -추가/삭제  │  
    Battle   │                                        │
│    Start                                            │
├────────────┴────────────────────────────────────────┤
│                     (푸터: 음소거 토글 — 아이콘 하나)      │
└─────────────────────────────────────────────────────┘
```

- **좌측 패널**: 참가자 입력 + Battle Start 버튼 + 실시간 탈락 순위 (게임 영역과 분리된 별도 페이지가 아닌, 게임 화면 좌측에 통합)
- **우측 메인**: 3D 스타디움 렌더링 영역 (화면의 약 75%~80%)
- **결과 표시**: 게임 종료 시 스타디움 위에 오버레이로 순위 표시 (별도 화면 전환 없음), 승리한 색상의 팽이 등장

### 모바일 (세로 화면)
- 상단: 인풋 패널 (접히는 아코디언)
- 하단: 게임 영역 (화면 대부분 차지)
- 탈락 순위: 하단 오버레이

## 2. 비주얼 스타일

### 컬러 & 톤
- **배경**: 짙은 다크 톤 `#0a0a12` — 팽이와 이펙트가 극적으로 부각
- **UI 텍스트/보더**: `rgba(255,255,255,0.15)` ~ `rgba(255,255,255,0.6)` — 은은한 화이트 계열
- **강조색**: 네온 계열 단색 (팽이 컬러와 동일 팔레트)
- **전체 무드**: 어둡고 미니멀, 여백 많음, 장식 요소 최소화

### 타이포그래피
- 산세리프, 가늘고 깔끔한 서체 (예: Inter, Pretendard)
- 폰트 웨이트: 300~400 위주, 강조 시 500
- 크기 계층을 최소화 (2~3단계만 사용)

### UI 컴포넌트
- 인풋 필드: 배경 투명
- 버튼: 미니멀 — 텍스트 기반 또는 얇은 아웃라인, 호버 시 은은한 글로우
- Battle Start 버튼만 유일하게 강조 (네온 글로우 보더)
- 불필요한 아이콘/장식 제거, 텍스트와 여백으로 구분

## 3. 팽이 디자인

### 디자인 레퍼런스: `top design.jpg`
- 단색 네온 색상 + 투명도가 있는 미니멀한 플라스틱/유리 질감

### 3D 팽이 스타일
- **형태**: 납작한 원반형 (CylinderGeometry 기반), 상단에 짧은 손잡이(축), 하단에 뾰족한 팁
- **재질**: `MeshPhysicalMaterial` — 투명도(opacity: 0.6~0.8), 유리/플라스틱 느낌
  - `transparent: true`
  - `roughness: 0.1` (매끄러운 표면)
  - `metalness: 0.1` (비금속, 투명 플라스틱 느낌)
  - `transmission: 0.3~0.5` (반투명 굴절)
- **컬러**: 참가자마다 단색 네온 컬러 랜덤 배정
  ```
  네온 그린    #39FF14  (opacity 0.7)
  네온 블루    #00BFFF  (opacity 0.7)
  네온 퍼플    #BF40FF  (opacity 0.7)
  네온 핑크    #FF6EC7  (opacity 0.7)
  네온 옐로우  #DFFF00  (opacity 0.7)
  네온 시안    #00FFEF  (opacity 0.7)
  네온 오렌지  #FF6700  (opacity 0.7)
  네온 레드    #FF003F  (opacity 0.7)
  ```
- **코어**: 팽이 중심에 구형 코어, 내부에서 빛이 은은하게 새어나옴 (PointLight 삽입)
- **스포크 단면**: 날카로운 블레이드 형태로 공격적 느낌
- **발광**: 팽이 자체 컬러와 동일한 네온 글로우 (하단 PointLight or emissive)

## 4. 시각 연출 & 이펙트

### 회전 표현
- 3D 메쉬가 Y축 기준으로 angularVelocity에 맞춰 실제 회전
- RPM이 낮아지면 기울어지며 세차운동(precession) 연출

### 파티클 & 트레일
- 팽이 밑면에서 빛나는 입자(Particle)가 뿜어져 나오며 궤적을 그림
- 속도가 빠를 때: 밝고 긴 궤적
- 속도가 줄어들면: 궤적이 짧아지며 흐려짐
- 빠르게 이동 시 발광 트레일 이펙트 (PointLight 기반 잔상)

### 충돌 이펙트
- 밝은 스파크(불꽃) 파티클 + 포인트라이트 플래시 + 화면 진동(shake)
- 어두운 배경에서 강렬하게 빛남

### 탈락 연출
- **Ring Out (장외)**: 스타디움 밖으로 포물선으로 날아가며 사라짐(페이드아웃) + 발광 폭발
- **Burst (분해)**: 강한 충격으로 팽이가 분해되며 사라짐 — 페이드 아웃

### 최종 승자 연출
- 마지막 팽이에 스포트라이트(SpotLight) + 줌인 + 컨페티 폭발

### 참가자 이름 라벨
- 각 팽이 위에 CSS2DRenderer 또는 Canvas 오버레이로 이름 명확하게 표시
- 탈락 시 순위와 함께 좌측 패널에 기록

### 스타디움
- 3D 원형 오목 메쉬 (벽 없는 CircleGeometry)
- 바닥에 움직이는 그라데이션 블룸/글로우 (Bloom/Glow, Nebula Effect) 효과가 있음
- 네온 컬러 테두리 발광
- 은은한 환경 조명으로 팽이가 잘 보이도록 설정

## 5. 사운드 디자인

- 팽이 회전음 (RPM에 따라 피치 변화 — 높을수록 고음)
- 충돌 시 금속성 타격음 (충돌 강도에 비례한 볼륨)
- 장외 탈락 시 "빠지는" 효과음 + 관중 반응
- 최후의 2~3개 남았을 때 배경음 긴장감 상승
- 승리 시 팡파레
- 음소거 토글 (스피커 아이콘 하나, 클릭 시 이미지/상태 변화, 셔플버튼 없음)

## 6. 반응형 디자인

- 프로젝터/대화면 대응 — 풀스크린 모드
