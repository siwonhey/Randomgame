## 1. 사이트 상세 분석

## 메탈블레이드 팽이 서바이벌 배틀 추첨
#### 컨셉
참가자 각각에 대응하는 팽이(베이블레이드)가 원형 스타디움 안에서 물리 시뮬레이션 기반으로 회전·충돌하며, 충돌 에너지를 이기지 못하고 스타디움 밖으로 튕겨 나가는 방식. 최후의 1개가 남을 때까지 배틀을 벌이며, 탈락 순서의 역순이 최종 순위가 된다.

#### 실제 팽이 배틀 물리 분석

##### 회전 역학 (Rotational Dynamics)
- **각운동량 보존**: 팽이는 초기 회전 에너지(½Iω²)를 갖고 시작하며, 마찰·공기저항·충돌로 에너지가 점진적으로 소실
- **자이로스코프 효과**: 고속 회전 시의 직립 안정성을 유지하다가, RPM이 떨어짐에 따라 세차운동(precession)을 거쳐 전도(topple)됨
- **접촉점 마찰**: 팽이 끝(tip)과 바닥의 마찰을 최소화하여 장외로 밀려날 때까지 회전력을 유지

##### 충돌 물리 (Collision Physics)
- 팽이는 시작 시 강한 힘으로 스타디움을 돌면서 서로 부딪혀야 하고(시작부터 너무 많은 팽이가 떨어지지 않도록 함),이후에도 스타디움에서 회전하며 서로 부딪히는 힘으로 하나를 제외한 모든 팽이들이 무조건 스타디움 밖으로 떨어지며 탈락해야 한다.
- **탄성 충돌**: 금속 재질의 특성을 반영하여 높은 반발 계수($e$)를 적용, 부딪히는 순간 멀리 튕겨 나가도록 설정
- **스매시 공격**: 빠르게 이동하는 팽이가 느린 팽이를 타격하면 큰 운동량 전이 → 스타디움 밖으로 밀어냄(Ring Out)
- **회전 방향 상호작용**: 같은 방향 회전 팽이끼리 충돌 시 미끄러지듯 접촉, 반대 방향이면 강한 마찰력으로 급격한 에너지 교환
- **무게중심 편향**: 무게가 한쪽에 치우친 팽이는 불규칙 궤적 → 예측 불가한 충돌 패턴 생성

##### 탈락 메커니즘 (2가지)
1. **Ring Out (장외)**: 충돌 충격으로 스타디움 바깥으로 튕겨나감 (주요 메커니즘)— 가장 극적인 탈락
2. **Burst (분해)**: 강한 충격으로 팽이가 분해되며 사라짐 — 페이드 아웃

##### 스타디움 역학
- **원형 오목한 경기장**: 중앙이 낮아 팽이가 자연스럽게 중앙으로 모이며 충돌 빈도 증가. **하지만, 팽이의 충돌 힘이 더 강해 장외 탈락이 가능하다.**
- **스타디움에 벽은 존재하지 않기 때문에** 팽이는 충돌 과정에서 스타디움 밖으로 사라지며 탈락한다.

#### 시뮬레이션 구현 설계

##### 물리 엔진 매핑
| 실제 물리 | 시뮬레이션 구현 | 엔진 |
|-----------|----------------|------|
| 팽이 회전 | 2D 원형 body + angular velocity (물리) / 3D 메쉬 회전 (시각) | Matter.js (물리) + Three.js (렌더링) |
| 회전 감속 | 프레임마다 angularVelocity *= decayRate(0.997~0.999) | 커스텀 |
| 스매시 | 충돌 순간 양측에 반대 방향의 강한 Force(충격량) 부여 | 커스텀 |
| 충돌 반발 | restitution 극대화 (1.0 이상 권장) | Matter.js |
| 스타디움 경사 | 중앙 방향 중력 벡터 또는 매 프레임 중심 방향 힘 적용(충돌 반발의 힘 보다 작아야 함) | 커스텀 |
| 탈락 판정 | 스타디움 원형 경계 밖으로 나가면 제거 | 커스텀 |
| 3D 렌더링 | Three.js WebGL 기반 3D 시각화 (어두운 배경 + 발광 이펙트) | Three.js |

##### 핵심 파라미터 (참가자별 랜덤 배정)
```
{
  mass: 0.8 ~ 1.2,           // 질량 — 무거우면 잘 밀리지 않음
  radius: 12 ~ 18,           // 크기 — 크면 충돌 확률 증가
  restitution: 1.0 ~ 1.2,   // 반발 계수 — 높으면 충돌 에너지가 증폭되며 더 멀리 튕김
  friction: 0.001, // 바닥 마찰 — 낮을수록 장외로 잘 미끄러짐
  angularDecay: 0.997 ~ 0.999, // 회전 지속력 — 높으면 오래 회전
  initialSpin: 1000 ~ 1400,   // 초기 RPM
  initialPosition: 원형 균등 배치, // 스타디움 가장자리에 균등 배치
  initialVelocity: 중앙 방향  // 발사 시 중앙을 향해 이동
}
```
> **공정성 보장**: 각 파라미터는 `crypto.getRandomValues()` 기반으로 결정.

> **시각 연출, 팽이 디자인, 사운드, UI 레이아웃** → `DESIGN.md` 참조

#### 게임 진행 플로우

```
[준비 단계]
  참가자 입력 → 입력에 맞춰 팽이 하나씩 스타디움에 자동 생성 (이름·색상·파라미터 배정)
  Battle Start 클릭 전에도 사용자가 입력하는 참가자가 첨삭되는 인원대로, 그에 맞게 실시간으로 Game Area에 이름표를 단 팽이가 하나씩 등장하거나 줄어들어야 함
  → 스타디움 위에 원형으로 배치

[카운트다운]
  3... 2... 1... GO!
  → 모든 팽이가 동시에 회전 시작, 중앙을 향해 발사

[배틀 단계]
  **발사 이후에도 팽이들이 지속적으로 서로 충돌·이동**(30초 안에는 승자가 겨뤄져야 함.)
  → 장외 시 탈락 판정(회전 정지 없음. 대부분 충돌로 인한 장외)
  → 탈락 순서대로 순위 기록 (최초 탈락 = 꼴등)
  → 사이드바에 실시간 탈락 순위 표시

[클라이맥스]
  최후의 2~3개 남았을 때 카메라 줌인 + BGM 전환
  → 최종 생존자 = 1등

[결과 단계]
  전체 순위표 표시 (1등~N등)
  → 컨페티 + 축하 연출
  → 결과 복사/공유/재추첨 옵션
```

## 2. 공통 핵심 요소 분석

모든 인기 사이트는 아래의 **3단계 UX**를 따릅니다:

```
[1단계: 입력] → [2단계: 실행] → [3단계: 결과]
  이름/항목 입력      버튼 클릭         결과 표시
  (최대한 간단하게)   (단 하나의 CTA)   (축하 연출)
```

**핵심 원칙**: 회원가입 없이, 설명 없이도 첫 추첨이 가능해야 함

---

#### A. 시각적 몰입감 (Visual Engagement)
> 상세 내용은 `DESIGN.md` 참조

#### B. 공정성과 신뢰 (Fairness & Trust)
- 암호학적 난수(`crypto.getRandomValues()`) 또는 물리 시뮬레이션으로 조작 불가 증명
- 과정이 눈에 보여야 함 — 블랙박스 결과보다 **과정 시각화**가 신뢰도를 높임

#### C. 제로 마찰 진입 (Zero-Friction Onboarding)
- 회원가입, 로그인, 설치 없이 즉시 사용
- 설명 없이도 직관적으로 사용 가능한 UI

#### D. 커스터마이즈 가능성 (Customizability)
- 항목 제거 옵션 (한번 뽑힌 이름 제거 후 재추첨)

#### E. 공유와 재사용성 (Shareability)
- URL 한 번으로 현재 설정 그대로 공유 가능
- 결과 복사/저장 기능

#### F. 반응형 및 멀티 디바이스 (Responsive Design)
> 상세 내용은 `DESIGN.md` 참조

## 3. 반드시 구현해야 할 핵심 요소

서비스 성공을 위해 없으면 안 되는 **필수 요소** (우선순위 순):

#### 1. 참가자 입력
- 이름 목록을 자유롭게 입력/추가/삭제
- 줄바꿈 또는 쉼표로 구분하여 붙여넣기 지원 (엑셀에서 복사-붙여넣기)
- 최대 30명 지원
- 입력 후 브라우저 저장(localStorage)으로 재사용 편의 제공

#### 2. 추첨 실행
- 버튼 하나로 즉시 시작되는 명확한 CTA
- 최소 10초, 최장 30초의 긴장감 있는 과정 연출(30초 내에 승자가 결정될 수 있도록 팽이끼리 부딪히며 충돌 힘을 가져야 함)
- 부드러운 물리 기반 감속 (linear가 아닌 easing)

#### 3. 결과 표시
- 당첨자 이름을 크고 명확하게 표시
- 결과를 닫고 재추첨하기 쉬운 UX
- 당첨자를 목록에서 자동 제거하거나 유지하는 선택 옵션

#### 4. 공유 기능
- URL로 현재 참가자 목록 상태를 공유
- 결과 텍스트 복사 버튼

> **UI 레이아웃, 시각 연출, 사운드, 반응형 디자인** → `DESIGN.md` 참조

## 6. 기술 구현 참고사항

### 물리 기반 방식 — 팽이 서바이벌 배틀
- **라이브러리**: `matter.js` (2D, 경량, 브라우저 최적화) 또는 `box2d-wasm` (정밀 충돌)
- **렌더링**: `Three.js` (WebGL 3D 렌더링, 어두운 배경 + 발광/스파크 이펙트, 원근감 있는 카메라) — 물리는 2D(탑다운), 시각은 3D
- **파티클 이펙트**: 충돌 스파크, 컨페티 → `tsparticles` 또는 커스텀 Canvas 파티클
- **사운드**: `Howler.js` (크로스 브라우저 오디오, 동시 재생, 볼륨/피치 제어)
- **성능 최적화**: 물리 스텝과 렌더링 분리 (fixed timestep), 오프스크린 팽이 간소화

### 공통 기술 스택 권장
- **난수**: `window.crypto.getRandomValues()` (Math.random 사용 금지)
- **애니메이션**: Web Animations API 또는 CSS Keyframes + requestAnimationFrame
- **컨페티**: `canvas-confetti` 라이브러리 (npm)
- **상태 저장**: localStorage (이름 목록, 설정값)
- **공유**: URL 파라미터 또는 URLSearchParams로 상태 직렬화
